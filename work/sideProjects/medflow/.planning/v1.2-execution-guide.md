# GSD v1.2 Execution Guide — Session Continuity

## INSTRUCCIONES PARA NUEVAS SESIONES

> **Si eres una nueva sesion leyendo esto:** Lee este archivo completo, luego lee `.planning/v1.2-session-state.json` para saber exactamente donde quedo la sesion anterior. Continua desde el ultimo checkpoint.

### 1. Como Retomar el Trabajo

```bash
# PASO 1: Leer el estado actual
cat .planning/v1.2-session-state.json | jq '.currentSession'

# PASO 2: Ver que tareas estan listas para ejecutar
cat .planning/v1.2-session-state.json | jq '.phases.phase1.tasks | to_entries[] | select(.value.status=="pending") | select(.value.dependsOn | length == 0 or all(.[]; . != "pending"))'

# PASO 3: Leer el plan de la fase actual
cat docs/MedNext\ v1.2/GSD-Plan/Fase-1-Sede-y-Unidad-Asistencial.md

# PASO 4: Leer la especificacion relevante
cat docs/MedNext\ v1.2/Especificaciones/Req-01-Sede-PuntoAtencion.md
```

### 2. Como Crear un Checkpoint

Antes de terminar una sesion o cuando se completa una tarea:

```bash
# Actualizar el estado con jq
jq '.currentSession.lastCheckpoint = now | .checkpoints += [{"timestamp": now, "phase": 1, "task": "BK-1.1.1", "status": "completed"}]' .planning/v1.2-session-state.json > tmp && mv tmp .planning/v1.2-session-state.json
```

O usar el script:
```bash
./scripts/v1.2-checkpoint.sh <task-id> <status> "<summary>"
```

### 5. Flujo por Tarea

```
1. Leer especificacion (Obsidian)
2. Leer codigo existente afectado (usar graphify para mapear)
3. Implementar cambio
4. Ejecutar tests
5. Si pasan: actualizar session-state.json (task → completed)
6. Si fallan: fix y reintentar
7. Al final: checkpoint
```

### 6. Reglas de Avance

- **NUNCA** saltar dependencias. Si BK-1.1.5 depende de BK-1.1.3 y BK-1.1.4, no ejecutar hasta que ambos esten completed.
- **SIEMPRE** actualizar `v1.2-session-state.json` despues de cada tarea.
- **SIEMPRE** crear checkpoint al final de cada sesion.
- Si una sesion se corta por longitud, la siguiente sesion lee el ultimo checkpoint y continua.
- Si una fase completa, marcar `status: "completed"` y desbloquear la siguiente fase.

### 9. Script de Checkpoint

Crear `scripts/v1.2-checkpoint.sh`:

```bash
#!/bin/bash
# Usage: ./scripts/v1.2-checkpoint.sh <task-id> <status> "<summary>"
TASK_ID=$1
STATUS=$2
SUMMARY=$3
STATE_FILE=".planning/v1.2-session-state.json"

# Update task status
jq --arg tid "$TASK_ID" --arg st "$STATUS" \
  '.phases.phase1.tasks[$tid].status = $st' \
  "$STATE_FILE" > tmp && mv tmp "$STATE_FILE"

# Add checkpoint
jq --arg ts "$(date -u +%Y-%m-%dT%H:%M:%SZ)" --arg tid "$TASK_ID" --arg st "$STATUS" --arg sm "$SUMMARY" \
  '.checkpoints += [{"timestamp": $ts, "task": $tid, "status": $st, "summary": $sm}] | .currentSession.lastCheckpoint = $ts' \
  "$STATE_FILE" > tmp && mv tmp "$STATE_FILE"

echo "Checkpoint saved: $TASK_ID → $STATUS"
```
